Managing Admins
Admins are not created from the dashboard. Admins must be created directly in Supabase.

Add a new admin

1
Open the correct Supabase project.
2
Go to Authentication - Users - Add user.
3
Create the email and password.
4
Run this SQL in that project's SQL editor:
update public.profiles set role = 'admin', full_name = 'Person Name Here', is_active = true where email = 'newemail@example.com';

Change admin email

1
Change it in Supabase Authentication.
2
Then run:
update public.profiles set email = 'newemail@example.com' where email = 'oldemail@example.com';

Change admin password

1
Supabase - Authentication - Users - send reset email or set password manually.

How To Roll Out A New Version

First-time rollout for a VA

  • Send them the correct Setup.exe for their company.
  • They install it once.
  • They open it from Start Menu or Desktop shortcut.
  • They log in and start work.

Future update rollout

  • Build new Thrive and Magik installers.
  • Upload each new installer into the correct company's desktop-downloads folder.
  • Update each dashboard's Update download URL.
  • If you want to force update, also set Minimum desktop version.
  • The installed app can then download the new installer and update itself.

Troubleshooting
ProblemWhat To Check First

VA cannot log inCheck VA is active, project assignment exists, and VA detail page agent events if any are showing.

Golden Rules
Never send the raw desktop-agent\dist\ThriveTracker.exe to VAs. That is internal only.
Never send a Thrive installer to a Magik VA or a Magik installer to a Thrive VA.
Never put installers in the private screenshots bucket.
Never put screenshots in the public desktop-downloads bucket.
Always build Thrive first, then Magik second.
Always upload the new installer file and update the dashboard download URL when you want installed apps to update.
If you forget anything, use this guide first. It is now the single reference for builds, rollout, storage, updates, and day-to-day operations.
