Build a VA (Virtual Assistant) Time Tracking & Monitoring System
Overview
Build a complete VA monitoring system similar to Hubstaff but simpler. It has TWO components:

Desktop Agent — A Windows app that VAs download and install. It tracks their work.
Web Dashboard — A website the admin uses to monitor all VAs.
Each company gets its own completely separate instance (separate Supabase project, separate database, separate users). There is no multi-company picker — each company is a standalone deployment.

Tech Stack (MUST USE THESE)
Desktop Agent: Python 3.11+, packaged as .exe using PyInstaller
mss — for fast screenshot capture
pynput — for keyboard and mouse activity tracking
pygetwindow — for active window title detection
Pillow — for image compression (JPEG)
requests or httpx — for API calls
tkinter — for the simple GUI (login window, timer controls, system tray)
pystray — for system tray icon
Backend + Database + Auth: Supabase (free tier)
PostgreSQL database
Supabase Auth for user login (admin + VA accounts)
Supabase REST API for data operations
Row Level Security (RLS) policies for data isolation
Screenshot Storage: Cloudflare R2 (free tier, S3-compatible)
Store screenshots as JPEG files
Organize by: /{user_id}/{YYYY-MM-DD}/{timestamp}.jpg
Web Dashboard: Next.js (React) deployed on Vercel (free tier)
Or alternatively: plain HTML/CSS/JavaScript deployed on Cloudflare Pages
Use Supabase JS client library for auth and data
Database Schema (Supabase PostgreSQL)
sql


-- Users table (extends Supabase auth.users)
CREATE TABLE public.profiles (
  id UUID PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE,
  email TEXT NOT NULL,
  full_name TEXT NOT NULL,
  role TEXT NOT NULL CHECK (role IN ('admin', 'va')),
  is_active BOOLEAN DEFAULT true,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
-- Projects
CREATE TABLE public.projects (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name TEXT NOT NULL,
  description TEXT,
  color TEXT DEFAULT '#3B82F6',
  is_active BOOLEAN DEFAULT true,
  created_by UUID REFERENCES public.profiles(id),
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
-- Project Assignments (which VAs are assigned to which projects)
CREATE TABLE public.project_assignments (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES public.profiles(id) ON DELETE CASCADE,
  project_id UUID REFERENCES public.projects(id) ON DELETE CASCADE,
  assigned_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(user_id, project_id)
);
-- Time Entries (start/stop work sessions)
CREATE TABLE public.time_entries (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES public.profiles(id) ON DELETE CASCADE,
  project_id UUID REFERENCES public.projects(id),
  started_at TIMESTAMPTZ NOT NULL,
  stopped_at TIMESTAMPTZ,
  duration_seconds INTEGER,
  is_manual BOOLEAN DEFAULT false,
  stop_reason TEXT CHECK (stop_reason IN ('manual', 'idle', 'app_close', NULL)),
  created_at TIMESTAMPTZ DEFAULT NOW()
);
-- Activity Logs (periodic activity snapshots, every 1 minute)
CREATE TABLE public.activity_logs (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES public.profiles(id) ON DELETE CASCADE,
  time_entry_id UUID REFERENCES public.time_entries(id) ON DELETE CASCADE,
  timestamp TIMESTAMPTZ NOT NULL,
  keystrokes_count INTEGER DEFAULT 0,
  mouse_clicks_count INTEGER DEFAULT 0,
  mouse_moved BOOLEAN DEFAULT false,
  activity_percent REAL DEFAULT 0,
  active_window_title TEXT,
  active_app_name TEXT
);
-- Screenshots
CREATE TABLE public.screenshots (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES public.profiles(id) ON DELETE CASCADE,
  time_entry_id UUID REFERENCES public.time_entries(id) ON DELETE CASCADE,
  project_id UUID REFERENCES public.projects(id),
  captured_at TIMESTAMPTZ NOT NULL,
  storage_url TEXT NOT NULL,
  storage_key TEXT NOT NULL,
  file_size_bytes INTEGER,
  activity_percent_at_capture REAL
);
-- Settings (admin-configurable)
CREATE TABLE public.settings (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  key TEXT UNIQUE NOT NULL,
  value TEXT NOT NULL,
  description TEXT,
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

Screenshot Capture Logic


When timer is running:
  Every [screenshot_interval_minutes] minutes:
    1. Capture all monitors using mss library
    2. Combine into single image if multi-monitor
    3. Compress to JPEG at [screenshot_quality]%
    4. Calculate file size
    5. Upload to Cloudflare R2: /{user_id}/{YYYY-MM-DD}/{HH-MM-SS}.jpg
    6. Save record to screenshots table via Supabase API
    7. If upload fails (no internet), queue locally and retry later
Activity Tracking Logic


Continuously while timer is running:
  Track keyboard:
    - Count every key press (just the count, NOT which key)
    - Reset count every 60 seconds
  Track mouse:
    - Count every click (left, right, middle)
    - Track if mouse is moving (boolean, sampled every second)
    - Reset count every 60 seconds
  
  Every 60 seconds:
    - Calculate activity_percent for this minute:
      - Divide the minute into 6 x 10-second slots
      - If there was ANY keyboard or mouse activity in a slot, it's "active"
      - activity_percent = (active_slots / 6) * 100
    - Get active window title using pygetwindow
    - Extract app name from window title
    - Upload activity_log record to Supabase
Idle Detection Logic


Continuously while timer is running:
  Track time since last keyboard OR mouse input
  If no input for [idle_timeout_minutes] minutes:
    1. Auto-pause the timer
    2. Update status to "Idle — paused"
    3. Mark the time entry's stop_reason as 'idle'
    4. Stop taking screenshots
    5. When user returns (any keyboard/mouse input detected):
       - Show popup: "You were idle for X minutes. Resume timer?"
       - If yes: create new time_entry, resume tracking
       - If no: keep timer stopped
Offline Queue


If any upload fails (screenshots, activity logs, time entries):
  1. Save to local SQLite queue database
  2. Every 30 seconds, check if internet is back
  3. If connected, upload queued items oldest-first
  4. Delete from local queue after successful upload
Settings Sync


On login and every 15 minutes:
  Fetch settings table from Supabase
  Update local configuration:
    - screenshot interval
    - screenshot quality  
    - idle timeout
    - etc.
  This way admin can change settings from dashboard and agents pick them up

Cloudflare R2 Setup
Create a bucket named va-tracker-screenshots
Generate API keys (Access Key ID + Secret Access Key)
Use S3-compatible API from the Python agent to upload
Use presigned URLs from the dashboard to display screenshots (so the admin can view them securely)
File path structure: {user_id}/{YYYY-MM-DD}/{HH-MM-SS}.jpg
Desktop Agent Distribution
Package with PyInstaller: pyinstaller --onefile --windowed --icon=icon.ico --name=ThriveTracker agent.py
The resulting ThriveTracker.exe is what VAs download
No installer needed — they just run the .exe
On first run, they enter: Server URL, email, password
After login, the app saves config locally and auto-connects next time
Security Considerations
All API calls use Supabase JWT tokens (auto-handled by Supabase Auth)
Screenshots are private in R2 — accessed only via presigned URLs
VAs cannot see each other's data (enforced by RLS)
Admin credentials should be strong
Desktop agent stores credentials in a local encrypted config file (not plain text)
Keyboard tracking counts keys only — it does NOT log which keys are pressed (privacy)
